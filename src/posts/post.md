# Использование React с GraphQL

Привет, бро. хочу немного рассказать про преимущества использования GraphQL как замену REST, в частности в React-приложениях

Дело в том что при использовании библиотеки Apollo, кардинально меняется подход к управлению состоянием. Если в случае Redux/Mobx у нас есть определенный стор, набор экшенов и редьюсеров, то в случае с Apollo, данные с каждого graphql запроса нормализуются и сохраняются в кэше. Что это значит - допустим, у нас есть следующий запрос:

```graphql
query getAllPosts {
  allPosts {
    title
    content
  }
}
```

Если бы мы использовали привычные библиотеки для управления состоянием, то при добавлении нового поста, нам бы пришлось создавать экшен, редьюсер, либо просто добавлять в массив постов, в случае с Mobx.

Чтобы повторить то же самое с Apollo, нам нужно будет записать новый пост в кэш
